# Transcript Review Policy

Status: current; no transcript ledger or records exist at bootstrap.

- Transcript-derived text is a retrieval aid, never project authority.
- Create claim-scoped review metadata only when current governance cites an
  exact transcript record.
- Absence means unclassified, not approved or rejected.
- Metadata authority effect is always `none`.
- Required metadata identifies the exact claim, source record/path/range,
  reviewer, UTC date, disposition, limitations, and promoted owner path if any.
- Promotion updates the real owner first; metadata may record the owner path
  afterward but never replace it.

Do not create transcript files, a ledger, or automation merely because this
policy exists.
