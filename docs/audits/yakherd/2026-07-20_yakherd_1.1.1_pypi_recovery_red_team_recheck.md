### Prior P1-1: closed

The preserved FAIL audit found that build, test, acceptance, and smoke-install
code ran in the same job that held `id-token: write`. The repaired
`.github/workflows/publish.yml` now creates two distinct jobs:

- `build` has only `contents: read`; it checks out the exact tag commit, binds
  tag/version/commit, verifies the reviewed hash chain, runs tests and clean
  acceptance, builds once, verifies both archives, smoke-installs both exact
  archives, and only then uploads them;
- `publish` has `needs: build`, so it cannot begin unless the entire
  unprivileged build job succeeds;
- only `publish` has `id-token: write` and the dedicated `pypi` environment;
- `publish` has no checkout, Python setup, shell command, build dependency,
  candidate script, test, or build backend step; and
- its only steps retrieve the named same-run artifact and invoke the publisher.

A static structure check found exactly one `id-token: write` grant, no OIDC
grant in the build job, no checkout/Python/run step in the publishing job, and
five action references that are all immutable 40-character commit SHAs.

The handoff is also bounded:

- upload uses official `actions/upload-artifact` v7.0.1 commit
  `043fb46d1a93c77aae656e7c1c64a875d1fc6a0a` once, after archive verification
  and both smoke tests;
- the two paths have `dist` as their least common ancestor, so the artifact
  contains the wheel and sdist filenames at its root;
- missing distributions are fatal, retention is one day, and no overwrite is
  enabled;
- modern upload-artifact archives are immutable, so a later job cannot append
  to or mutate this named artifact;
- download uses official v8.0.1 commit
  `3e5f45b2cfb9172054b4087a40e8e0b5a5461e7c`, names the one artifact, and omits
  cross-run credentials and identifiers, which scopes retrieval to the current
  repository and current workflow run;
- downloading one named artifact directly to `dist` recreates the layout
  expected by the publisher; and
- the PyPA v1.14.0 publisher remains pinned to immutable source commit
  `cef221092ed1bacb1cc03d23a2d87d1d172e277b`, whose matching GHCR image tag
  exists and is selected from `github.action_ref`.
